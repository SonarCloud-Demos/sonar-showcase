# Documentation Locations

**After a successful `mvn clean install`, documentation is available at:**

---

## Backend - JavaDoc

**Location:** `backend/target/site/apidocs/`

**Index Page:** `backend/target/site/apidocs/index.html`

**Open in browser:**
```bash
open backend/target/site/apidocs/index.html
```

**What's included:**
- All Java classes and interfaces
- Method documentation with `@param` and `@return` tags
- Class-level documentation
- Package organization

---

## Frontend - TypeDoc

**Location:** `frontend/target/site/typedoc/`

**Index Page:** `frontend/target/site/typedoc/index.html`

**Open in browser:**
```bash
open frontend/target/site/typedoc/index.html
```

**What's included:**
- React components
- TypeScript services and utilities
- Custom hooks
- Type definitions
- Function documentation

---

## Quick Access Commands

### View Both Documentation Sets

**From project root:**
```bash
# Backend JavaDoc
open backend/target/site/apidocs/index.html

# Frontend TypeDoc
open frontend/target/site/typedoc/index.html
```

### Or use relative paths:
- Backend: `./backend/target/site/apidocs/index.html`
- Frontend: `./frontend/target/site/typedoc/index.html`

## Published Documentation (GitHub Pages)

If GitHub Pages is configured (see [GITHUB_PAGES_SETUP.md](./GITHUB_PAGES_SETUP.md)), documentation is automatically published online:

**Main Index:**
- URL: `https://your-username.github.io/sonar-demo/`
- Landing page with links to both documentation sets

**Backend JavaDoc:**
- URL: `https://your-username.github.io/sonar-demo/backend/`
- Same content as local `backend/target/site/apidocs/`

**Frontend TypeDoc:**
- URL: `https://your-username.github.io/sonar-demo/frontend/`
- Same content as local `frontend/target/site/typedoc/`

**Key Difference:**
- **Local docs** (`target/`): Generated on your machine, not in git
- **Published docs** (GitHub Pages): Generated by CI/CD, served publicly, stored in `gh-pages` branch

---

## Notes

- Documentation is **generated during build** - run `mvn clean install` to regenerate
- Documentation is in `target/` directories - these are excluded from git (see `.gitignore`)
- Both documentation sets are HTML-based and can be viewed in any web browser
- Documentation reflects the current state of the codebase at build time
- **GitHub Pages** automatically publishes docs on every push to main branch (if configured)
